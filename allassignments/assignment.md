1. What is Next.js?

Next.js is a React-based framework that helps build fast websites using server-side rendering and easy routing.

2. Difference between React and Next.js

React is a UI library; Next.js is a full framework with routing, SSR, APIs, and better performance features.

3. What is Server-Side Rendering?

It means the page is created on the server before sending it to the browser.

4. What is Client-Side Rendering?

It means the browser loads the page and then React builds the UI using JavaScript.

5. How does routing work in Next.js?

Pages are created automatically based on the folder and file names inside the pages or app directory.

6. What is a dynamic route?

A route that changes based on a value, like /user/1 or /product/123.

7. What are API Routes in Next.js?

Special files inside pages/api or app/api that let you create backend APIs inside your Next.js project.

8. What is getStaticProps used for?

It fetches data at build time and creates a static page that loads super fast.

9. What is the purpose of the app directory in Next.js?

It is the new folder for building Next.js apps using the App Router with layouts, server components, and better structure.

10. What is the use of the Image component in Next.js?

It loads images faster by optimizing size, format, and quality automatically.
